# 🧙‍♂️ BG3 Party Builder

> A DevOps portfolio project — a serverless AWS application that recommends Baldur's Gate 3 companions based on your playstyle.

**Live:** https://lariwerneck.github.io/bg3-party-builder

---

## What it does

You describe how you want to play — "I want to be manipulative and cruel" or just pick a preset like *The Hero* — and the app recommends which companions will have the highest approval with your character, backed by real approval data scraped from the BG3 wiki.

---

## 🛠️ Architecture

```
User → GitHub Pages (frontend)
          ↓ fetch
     API Gateway → Lambda (Python) → S3 (JSON data)
```

| Layer | Tech |
|---|---|
| Frontend | HTML, CSS, JavaScript — hosted on GitHub Pages |
| API | AWS API Gateway (REST) |
| Backend | AWS Lambda (Python 3.12) |
| Data | AWS S3 (companions.json, approval_events.json) |
| Infrastructure | Terraform |
| CI/CD | GitHub Actions |
| Auth | OIDC (no AWS keys stored as secrets) |
| Monitoring | AWS CloudWatch + AWS Budgets alert |

## 🔗 API endpoints

| Endpoint | Description |
|---|---|
| `GET /party?style=hero` | Returns recommended party for a playstyle |
| `GET /party?style=i want to be manipulative` | Also accepts free-text descriptions |
| `GET /companion?name=Shadowheart` | Returns companion details + approval events |
| `GET /playstyles` | Lists all available playstyles |

**Base URL:** `https://csicwu75lc.execute-api.us-east-2.amazonaws.com/dev`

---

## 🎲 How the party builder works

1. **Scraper** — collects companion pages and approval events from [bg3.wiki](https://bg3.wiki) via the MediaWiki API, runs weekly via GitHub Actions
2. **Playstyle detection** — matches free-text input against keyword sets for 6 playstyles (hero, villain, rogue, scholar, warrior, diplomat)
3. **Companion scoring** — cross-references playstyle keywords against real approval events from the wiki, scores each companion, returns top 3
4. **Fallback** — uses companion likes/dislikes metadata when event data is sparse

## ☁️ Deploying

### Prerequisites
- AWS CLI configured
- Terraform >= 1.5
- GitHub repo with secrets: `AWS_ROLE_ARN`, `ALERT_EMAIL`, `DATA_BUCKET`

```bash
# Create state bucket (once)
aws s3 mb s3://bg3-party-builder-tfstate --region us-east-2

cd terraform
terraform init
terraform apply -var="alert_email=your@email.com"
```

Push to `main` triggers the full pipeline automatically.

---

## 📌 CI/CD pipeline

```
push to main
    │
    ├── Test Lambda     → runs handler.py against sample data
    ├── Terraform Validate → fmt check + validate
    │
    └── Deploy to AWS (only on main)
            ├── terraform plan
            └── terraform apply
```

Every Sunday at 03:00 UTC the scraper runs and uploads fresh data to S3.

## 🚀 DevOps highlights

- **Infrastructure as Code** — all AWS resources defined in Terraform, zero manual console clicks
- **Serverless** — Lambda + API Gateway, ~$0/month at portfolio scale
- **OIDC auth** — GitHub Actions authenticates to AWS via OpenID Connect, no long-lived credentials stored
- **Remote state** — Terraform state stored in S3, enabling consistent deploys from any machine
- **Budget alert** — AWS Budgets configured to notify at $1/month threshold
- **Weekly data pipeline** — scheduled GitHub Actions workflow keeps wiki data fresh

## 🎲 Como funciona o construtor de grupos

1. **Scraper** — coleta páginas de companheiros e eventos de aprovação do [bg3.wiki](https://bg3.wiki) via API do MediaWiki, executado semanalmente via GitHub Actions
2. **Detecção de estilo de jogo** — compara a entrada de texto livre com conjuntos de palavras-chave para 6 estilos de jogo (herói, vilão, ladino, erudito, guerreiro, diplomata)
3. **Pontuação de companheiros** — cruza as palavras-chave de estilo de jogo com eventos de aprovação reais da wiki, pontua cada companheiro e retorna os 3 melhores
4. **Fallback** — usa metadados de curtidas/descurtidas de companheiros quando os dados de eventos são escassos
